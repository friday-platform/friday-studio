and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [0.1.5] - 2026-05-08

### Changed

- Playground Svelte/Vite warnings cleared. Melt UI builders (`collapsible`, `dialog`, `dropdown-menu`, `tooltip`, `tree`) wrap option objects in `untrack(...)` so prop destructures stop tripping `state_referenced_locally`; `useCredentialConnect` accepts a `() => providerId` getter so the read is reactive at the call site. Dead scoped CSS, a redundant `role="banner"`, and the unused `signalDetails` plumbing through `job-index-sidebar` removed alongside. (#235)

### Fixed

- Chat send no longer aborts its own streaming response. `user-chat.svelte` was handing `ChatImpl` the same `$state` array the resume `$effect` was tracking; the AI SDK's in-place push re-triggered the effect, whose cleanup called `instance.stop()` and killed the live SSE. Now clones the array before construction and wraps the trailing-user check in `untrack(...)`. Regression from #230. (#239)
- Chat debug page no longer wedges the daemon gate on "Connecting to daemon…". `shortJson()` / `fullJson()` called `JSON.stringify(undefined)` (which returns `undefined`) and then `.length` on it for `tool-*` parts in `output-error` state that never received an `input` — the thrown `TypeError` aborted the parent reactive update so `daemon-gate.svelte` never swapped to its loaded branch. Both helpers fall back to `String(value)`. (#238)

## [0.1.4] - 2026-05-08

Durable Human-in-the-Loop, artifact-ref handoff, and per-action validation contracts (#225) are the headline; chat connection plumbing got a major cleanup pass too (#230). Read **Breaking changes** before upgrading.

### Breaking changes

- **`scratchpad` storage and tools removed.** Agents that need chunked working state should declare a `memory.own` store typed `short_term` and call `memory_save`/`memory_read` instead. Deletes the `packages/adapters-memory/` package, the `/api/scratchpad` route, `ScratchpadWriteEventSchema`, the `scratchpad_*` entries in `PLATFORM_TOOL_NAMES`, and the `atlas-cli inspect --kind scratchpad` arm. Existing daemons drop the orphaned `KV_SCRATCHPAD` bucket via the new `m_20260507_120000_drop_scratchpad_kv` migration. (#225)
- **`outputTo` LLM actions must finish via the `complete` tool.** Free-form `outputTo` actions now inject a mechanical `complete` tool contract even when `outputType:` is absent; tool-using actions run with `toolChoice: required` plus a hard post-call `complete` requirement. Authors who relied on plain prose endings will see actions fail validation until they emit through `complete`. Closes a class of empty-`response` regressions where `record_validation` (or another final tool) would persist an empty stub. (#225)
- **`OutputValidator` / `validateOutput` engine option removed.** `FSMEngineOptions.validateOutput`, the `legacyValidatorShim`, and the `OutputValidator` / `LLMOutputValidationResult` types are gone — `validate: external` now exclusively routes through the delegate-driven `@friday/judge-agent`. `packages/hallucination` shrinks by ~3,170 LOC; the `retryOnFail` field is dropped from `validate:` (wrap actions in an FSM-level retry pattern instead). (#225)
- **`validate.threshold` and `successSignal` schema fields removed.** `ValidateStrategySchema`'s object-form `threshold` enum was parsed but never read; `SessionSummarySchema.successSignal` plus `SuccessSignalSchema` were reserved-but-never-populated. Workspaces that hand-coded either field will fail validation. (#225)
- **`LocalSessionHistoryAdapter` deleted** (~470 LOC); production has wired `JetStreamSessionHistoryAdapter` exclusively since 0.1.2. Consumers should switch to the JetStream adapter or an inline in-memory mock for tests. (#225)

### Migration

- **`ELICITATIONS` JetStream stream now requires `allow_msg_ttl: true`.** Daemons that ran an earlier broken migration created the stream without this flag, so on NATS 2.11+ every `request_tool_access` and `request_human_input` publish failed silently with "per-message ttl is disabled". A pre-flight `bootstrapElicitationsStream(nc)` runs at daemon boot ahead of HTTP route mount and self-heals misconfigured streams in-place via `streams.update`. The adapter no longer self-creates the stream — it surfaces a clear "re-run migration" error if the bootstrap didn't run. (#225)
- **`SESSION_EVENTS` JetStream `duplicate_window` raised from 2m default to 24h.** Long-running FSM jobs (auto-triage > 2 min, chat-flip ~150s) outran the 2-minute dedup window: re-published stable-msg-id step events landed as new entries, the reducer appended duplicate `AgentBlock`s, and status derivation surfaced sessions as `active` post-completion. Existing streams reconcile automatically via `streams.update`; the reducer also got a defense-in-depth no-op for `(stepNumber, agentName)` duplicates from any pre-fix events still in stream history. (#225)

### Added

- **Durable Human-in-the-Loop (HITL) primitives.** Two new platform MCP tools — `request_tool_access(toolName, reason)` and `request_human_input(question, options?, multi_select?)` — turn LLM-blocking events into durable elicitations the user answers from the Activity feed. Backed by a new `packages/core/src/elicitations/` module (Zod schema + JetStream adapter; `ELICITATIONS` stream + `ELICITATION_STATUS` KV), HTTP routes at `/api/elicitations`, a 60s pending→expired sweeper with read-time derivation, and CAS-fenced terminal transitions so concurrent answers race-safely. Allow-always tool-access grants persist in workspace-scoped KV and short-circuit before creating new elicitations. `expiresAt` derives from job-scoped `config.timeout`, falling back to `DEFAULT_ELICITATION_TTL_MS` (30 min). (#225)
- **Activity page in agent-playground.** New SvelteKit routes at `/activity` (global) and `/workspaces/:id/activity` (scoped) list pending + recent elicitations; live updates via workspace-scoped SSE on the scoped page, 5s polling on the global page (deliberately avoids opening one EventSource per workspace, which exhausted the per-origin browser request pool). Filters by status / workspace / kind; expired elicitations render read-only; sidebar pending-count badges share the Activity TanStack cache. Repeated `request_human_input` prompts (numbered Actions shape) render as structured choice rows instead of one enormous radio group. (#225)
- **FSM action outputs persisted as artifact references mid-session — the supervisor flip.** `outputTo` documents are written through `ArtifactStorage` immediately after each action completes (not just at session-complete), so downstream `inputFrom` steps consume compact `{artifactId, summary}` refs and the next LLM call hydrates content via `expandArtifactRefsInInput`. SSE `job-complete` and the workspace-chat job-tool now return `{success, sessionId, status, artifactIds, summary}` instead of full `Document[]`; the chat-supervisor's next-turn input drops from full job output to refs + summary, with `parse_artifact(<id>)` available for selective fetch. The 50-email auto-triage scenario the plan was built around now returns ~"Processed 50 emails, 45 actions, 5 flagged" + 50 artifact IDs to chat instead of inflating the supervisor context. Legacy `output: Document[]` still emitted alongside for back-compat. (#225)
- **Per-action `tools:` and `skills:` allowlists.** `LLMAction.tools` / `LLMAction.skills` (mirrored on `AgentAction`) were `@experimental` and silently ignored — now wired as runtime narrowing filters on top of `resolveVisibleSkills` and the workspace MCP catalog. `tools` narrows MCP-server tools only (platform built-ins like `memory_save`, `artifacts_*`, `fs_*`, `bash`, `request_human_input`, `load_skill`, `delegate` are ambient and auto-injected); `skills: []` opts out of `load_skill` entirely; `skills: ["a","b"]` whitelists within the resolved set. (#225)
- **`delegate` tool with workspace + per-job budget enforcement.** FSM `type: llm` actions can opt into a sub-agent fan-out via `tools: [..., "delegate"]` — the engine constructs a synthetic `UIMessageStreamWriter` that bridges chunks into the FSM signal's `onStreamEvent`. New `delegation: { max_depth, max_steps_per_call, max_output_tokens, max_input_tokens, max_wall_time_ms, max_cost_usd }` block at workspace + per-job level promotes the previously-hardcoded `CHILD_STEP_BUDGET` / `CHILD_MAX_OUTPUT_TOKENS` constants to config-driven and adds wall-clock, input-token, and depth enforcement. On exhaustion, delegate returns `{ ok: false, reason: "budget_exhausted: <which>" }`. `delegate/` moved from `workspace-chat/tools/` to `@atlas/core/delegate` so chat and FSM share one import. (#225)
- **`permissions: { dangerouslySkipAllowlist }` block at workspace + per-job level.** Mirrors Claude Code's `--dangerously-skip-permissions` with three-level precedence merged via `resolvePermissions`: job > workspace > daemon `FRIDAY_DANGEROUSLY_SKIP_PERMISSIONS` env var. Each level can override its parent in either direction. When bypass resolves true, FSM `buildTools` skips per-agent tool narrowing and `request_tool_access` short-circuits with `{granted: true, reason: "bypass"}`. (#225)
- **Validation strategy contract (`validate:` field + workspace/job defaults).** Authors pick `skip | self | external | auto` (or an object form `{ strategy, skill, agent }`) per action; `auto` (default) routes through a pure-function classifier that picks `skip` for read-only/structured emits and `self` for mutating-tool/prose-emitting actions. `external` fires only on explicit opt-in — most workloads pay zero validation cost. `validate: self` composes the new `@friday/validating-llm-outputs` system skill into the prompt and auto-injects a `record_validation` platform tool; `validate: external` invokes a `@friday/judge-agent` delegate that sees the action output + a tool-call manifest with `resultArtifactId` for scrubber-lifted results. Per-MCP-server `validation: "read-only" | "mutating"` overrides honor author intent over the regex defaults. Every `step:complete` event now carries a `validation: { strategy, verdict?, issues?, skipReason? }` field, and `FSMEngine.initialize()` emits one info log per `(state × action)` summarizing the statically-resolved decision so authors can grep daemon logs without bisecting precedence chains. (#225)
- **Ephemeral artifact lifecycle + grace-window sweeper.** Artifacts and narrative-memory entries gain a `lifecycle: { kind, boundTo? }` field (`durable` | `ephemeral`). FSM action outputs from non-terminal states + `notes` (short_term memory) default to ephemeral, bound to session; outputs from terminal states + `memory` (long_term) stay durable. New `apps/atlasd/src/sweepers/artifacts-sweeper.ts` runs hourly (configurable via `FRIDAY_SWEEPER_INTERVAL_MS`) and at expiry promotes referenced artifacts to durable (memory text contains the id, `aiSummary.keyDetails` URL points at it) or deletes them otherwise. Workspace.yml overrides via `jobs.<name>.artifacts.{ephemeral,default_grace}` and `memory.own[].ttl`. (#225)
- **Scrubber lifts large text/JSON tool results to artifacts.** Text/JSON tool results above 8 KB now lift to artifacts before reaching the LLM message buffer (binary lifting was already shipped). Lifting moved off the MCP-tool-result boundary onto the persistence layer (`liftToolResultsForPersist`) — the producer LLM operates on inline bytes, persistence and cross-consumer handoff stay compact. Marker text shape unchanged so G2 marker rendering, session events, agentBlocks, and `inputFrom` keep working. Agent-playground tool-call cards auto-render lifted-artifact markers via the existing artifact preview. (#225)
- **Retrieval-gated artifact injection in FSM/chat prompts.** Both chat and FSM `type: llm` actions now inject a `<retrieved_content provenance="..." origin="..." fetched_at="...">` block carrying recent session artifacts (up to 10 per session, queried via the new `ArtifactStorageAdapter.listBySession`). Each block carries summary + artifactId so the LLM can `parse_artifact` for full content. (#225)
- **Async `aiSummary` off the critical path + fast-path from terminal `action.summary`.** When the terminal action declares `summary:` AND has structured `outputType:`, runtime builds `SessionAISummary` synchronously from the declared summary + leaf-field-derived `keyDetails` and skips `generateSessionSummary` entirely. Otherwise `generateSessionSummary` runs as a detached promise after `sessionStream.finalize()` — SSE `job-complete` responds immediately with the action.summary fallback; the LLM-generated summary writes asynchronously and emits a `session:summary` event for live subscribers. Saves 1–2s per job. (#225)
- **Session provenance + per-step token usage.** `SessionSummary` gains optional `parentSessionId` / `parentEventId` populated from chat-spawned-job, delegate-spawned-child, and signal-trigger paths so the daemon can walk a chat→job→child tree from the root. `step:complete` events now persist `usage: { inputTokens, outputTokens, cacheReadTokens, cacheWriteTokens, model }`; `AgentBlock.usage` aggregates from `step:complete.usage` so workspace-chat token usage stops being silently dropped in nested-agent flows. (#225)
- **Python user-agents receive structured FSM input via `ctx.input`.** FSM action input (including `inputFrom` artifact-ref hydration) is threaded into the Python subprocess agent execution path, and active `FRIDAY_NATS_URL` is forwarded into subprocess registration so harness daemons on random NATS ports can run user agents. User-agent `agent.py` paths can call `ctx.tools.list()` / `ctx.tools.call()` against host-injected platform tools, including `request_human_input`. New `writing-friday-python-agents` and `repairing-workspaces-agents` system skills encode the production-derived authoring + repair runbooks. (#225)
- **Live-daemon QA harness + first-principles eval suite.** New `tools/qa/live-daemon/harness.ts` spawns a fresh `atlasd` against an isolated `FRIDAY_HOME` + isolated NATS server, registers fixture workspaces, fires signals, parses SSE, and walks raw session events for token-usage aggregation. Twenty-plus first-principles scenarios cover scrubber lift, platform-tool parity, ephemeral lifecycle, delegate budgets, retrieval injection, all three HITL terminal states, supervisor-flip compactness, structured `validate: self` verdict shape, and Python user-agent inputFrom + tool calls. Replaces the obsolete phase-numbered runner + `inbox-corpus` fixtures + committed result artifacts. (#225)
- **Sanitized global elicitation SSE feed.** New `GET /api/elicitations/stream` emits a sanitized summary schema for `elicitations.>` — only safe metadata, no full envelopes. The app root mounts a single subscriber that refetches affected list caches, replacing both the old global 5s polling and the per-workspace EventSource fanout that #229 had to disable. Workspace chat/Activity streams continue to use workspace-scoped full envelopes for inline HITL forms. (#230)
- **Three deterministic ops in the bundled hubspot agent: `create-note`, `read-ticket`, `noop`.** Each is recognised by `HubSpotOperationSchema` before dispatcher fall-through to the LLM. `create-note` posts a CRM Note via the existing `batchCreateCrmObjects` helper using the standard notes-tickets association id (228); `read-ticket` fetches the ticket via the SDK and returns structured `data.{ticketId, subject, body, priority, pipelineStage, companyId, contactIds, threadId, transcript, ...}` while keeping `response` a one-line human status (explicitly NOT structured data in markdown form, which would re-create the regex-scraping fragility); `noop` is a success-with-no-side-effects sentinel for empty-cron-tick paths. Drops the bucketlist-cs auto-answer cron runtime by ~30s/tick and lets five downstream user agents read from `data.*` instead of regex-scraping prose. (#220, #228)

### Fixed

- **HITL waiter races and KV-lookup correctness.** A transient missing KV read masqueraded as a pending terminal result and unblocked the caller with bogus state — waiter now distinguishes "row missing" from "row pending" and only treats a real terminal status (or a 5s deadline grace window) as the answer. Subscribe-then-flush-then-reread closes the fast-answer race. `request_human_input` waits for terminal state instead of returning immediately with `pending_user_approval`. `expirePending` flipped to publishes-then-CAS (was CAS-then-publish, where a publish failure left KV `expired` with no stream witness and the transition never reached SSE subscribers). Elicitation publishes carry `Nats-Msg-Id` (`elicitationId:status`) so retried publishes within the 2m broker dedup window collapse. `list()` materializes keys before iterating to work around the nats.js v2.29 issue where `kv.get(key)` inside `for await (const key of kv.keys())` terminates the consumer early. (#225)
- **`request_tool_access` rejects unknown tool names without creating user-facing elicitations.** The runtime-visible tool catalog is now threaded through `ToolScope.toolCatalog` so the permission tool can validate against real platform/runtime tool names before the bypass or elicitation branches; unknown names return structured discovery guidance. (#225)
- **Elicitation SSE stream now requires `workspaceId`.** `GET /api/elicitations/stream` previously broadcast full envelopes on a global live feed; SSE now requires `workspaceId` so cross-workspace data doesn't leak. The agent-playground sidebar and global Activity page switched from per-workspace EventSource fanout (which exhausted the per-origin browser request pool) to 5s polling. (#225)
- **`request_human_input` UI nesting + repeated-prompt rendering.** Live HITL chat freeze occurred because `request_human_input` was nested under a running parent job tool and hidden inside the collapsed parent's tool burst — user-action detection now recurses into tool children. Repeated per-item prompts render as structured choice rows instead of one enormous radio group; the backend dedups identical pending elicitations within the same session/action/question/options scope. (#225)
- **Active platform MCP / session runtimes excluded from idle reaping.** Blocked HITL runs spent waiting on user input were getting cleaned up as idle — `WorkspaceRuntime.sessions` only contains finalized sessions, so idle-reap checks now consult `activeAbortControllers` for in-flight runs. (#225)
- **Activity HITL elicitations no longer fall back to Activity-only CTAs.** Newly-created `request_human_input` prompts could appear in chat before their persisted elicitation reached the TanStack list cache, so `human-input-tool-card.svelte` skipped inline rendering. The chat pages now subscribe to the same workspace elicitation SSE feed as Activity, and the card renders the form as soon as tool args are available while the persisted elicitation syncs in. (#230)
- **SvelteKit daemon proxy leaked upstream sockets when SSE consumers disconnected.** The proxy was returning the upstream daemon `Response.body` directly, which does not reliably cancel the upstream daemon request when the downstream browser closes — killing the client through `:5200` left a Vite→daemon `:8080` ESTABLISHED socket alive. New `proxyAbortableBody` wraps the upstream body in a `ReadableStream` whose `cancel()` and `request.signal` `abort` listener both call an upstream `AbortController.abort()`. Companion client-side cleanup: `user-chat.svelte` stops AI SDK chat streams on route teardown; session/elicitation queries propagate TanStack abort signals; daemon health polling is now non-overlapping and teardown-aware. (#230)
- **Server-side MCP transports outlived completed tool calls.** `lsof -nP -iTCP:8080` showed many daemon PID self-connections after `agent_web`/tool-call activity even with no active workspace sessions: `StreamableHTTPClientTransport.close()` aborts the client side but does not send the MCP DELETE lifecycle request. `AgentOrchestrator` now calls `terminateSession()` before closing each client transport, sets `Connection: close` on internal MCP fetches, and exposes `releaseSession(sessionId)` invoked from the workspace runtime when a session reaches a terminal state (idle sweep shortened to 5 min as fallback). `AtlasDaemon.cleanupAgentSession` / `cleanupPlatformSession` await `transport.close()` and `server.stop()` explicitly. (#230)
- **Aborted signal waits leaked NATS subscriptions; cascades cancelable before a session exists.** `awaitSignalCompletion` now accepts an `AbortSignal` and unsubscribes on whichever resolves first (was: only `iter.next()` vs timeout). New `signals.cancel.<correlationId>` subject + `publishSignalCancellation` helper let `CascadeConsumer` honor cancellation before dispatch and abort in-flight cascades whose correlation arrives mid-flight. Signal routes call `nc.flush()` after subscribing to the response subject so fast replies cannot race subscriber readiness — core NATS `subscribe` can return before the server has registered interest. (#230)
- **MCP streaming responses released before their bodies finished.** `AtlasDaemon.handlePlatformMcpRequest` released `activeRequests` in `finally` as soon as `transport.handleRequest()` returned, but `@hono/mcp` can return a streaming SSE `Response` whose body is still owned by an in-flight MCP request — stale-session cleanup could close the transport while that body was still being written. New `trackMcpResponseLifetime` wraps the response body in a `ReadableStream` that releases only on body close/error/cancel. (#230)
- **Activity deep-links surface the target; daemon-gate stops flashing during long streams.** `activity-view.svelte` reconciles filter state on deep-link so `Pending` doesn't hide answered/declined/expired/cross-workspace elicitations. `daemon-health.svelte.ts` tracks first successful connection separately from current health and requires repeated failures within a grace window before marking an established connection down; `daemon-gate.svelte` keeps content mounted and renders a non-blocking inline reconnect banner for transient failures. (#230)
- **`fs_write_file` resolves relative paths against the workspace dir.** Was using `process.cwd()`, so chat-driven jobs wrote files into the daemon's launch dir instead of `~/.atlas/workspaces/<workspaceId>/`. Added to `SCOPE_INJECTED_PLATFORM_TOOLS` so the runtime injects `workspaceId` on every call; absolute paths bypass entirely. Also creates missing files instead of treating ENOENT as fatal during its pre-write `stat` check. (#225)
- **`resolveAgentType` falls through to the system-agent registry for bundled agents.** Bundled `type: atlas` agents like `workspace-chat` and the new `judge-agent` don't appear in `this.config.workspace.agents`; the callback returned `undefined` and the validation classifier's `user/atlas → skip` rule never fired for them. Now correctly resolves `workspace-chat` as `atlas → skip` (no validation overhead on the chat path). Sibling fix: `user-invocable: false` skills are filtered from the user-facing skill catalog so `@friday/validating-llm-outputs` no longer leaks into `load_skill` autocomplete (direct `SkillStorage.get` calls remain unaffected). (#225)
- **Structured `outputType:` + `validate: self` no longer breaks `complete`-tool emission.** Auto-injection of `record_validation` when decision === `self` flipped `toolChoice` from `{type: "tool", toolName: "complete"}` to `"auto"`, letting the LLM emit free-form prose. When `outputType:` AND decision resolves to `self`, runtime now skips both `record_validation` injection AND `composeValidationBlock` skill body composition — the structured schema IS the validation contract; verdict is implicit pass on successful `complete` emission. Companion fix: empty `result.text` recovered by scanning `steps[]` backward for the latest text-typed content part when the LLM ends on a tool call. (#225)
- **Activity page 500 from `node:path` browser bundling.** Adding adapter exports to `packages/core/src/elicitations/mod.ts` made the umbrella transitively import `@atlas/logger → packages/logger/src/paths.ts → node:path`, which Vite externalizes for browser. New `@atlas/core/elicitations/model` subpath only re-exports the schema + types (zod-only); agent-playground client code switched to it. (#225)
- **JetStream session-history adapter robustness.** `ensureStream` typed catch (was: bare `catch {}` after `streams.update` masked permanent failures by falling through to `streams.add` and hiding the real error); `get()` no-progress guard breaks with a warn after a single empty fetch so a degraded stream doesn't lock up the read path; consumer-name suffix uses `crypto.randomUUID()` instead of `Date.now()` (which collided on same-millisecond reads, throwing `consumer-already-exists` and silently returning null). ### Fixed

- **Chat handler buffer-capture race.** The handler captured `ownBuffer = streamRegistry.getStream(chatId)` *after* awaiting `thread.subscribe` and `ChatStorage.appendMessage` — in that 10–50ms window, a follow-up POST could replace the chatId-keyed buffer, causing the handler to write turn 1's late chunks into turn 2's buffer. Now stashes the buffer on `Message.raw.turnBuffer` at dispatch and reads it back deterministically; non-web adapters (Slack / Telegram / Teams) that never create a buffer no longer log spurious `stream_event_dropped` warnings. (#206, follow-up to #192/#193)
- **Cross-turn event leak in chat stream registry.** Two cooperating bugs let a stopped turn's late events corrupt the next turn's stream: `finishStream(chatId)` was called in `finally` without identity verification, so a fresh buffer for the same chatId was flipped to inactive by the prior turn's cleanup; `appendEvent` looked up buffers by chatId only, so late events from an aborted turn (subprocess emit, NATS reconnect-buffer flush, streamSub drain) landed in whatever buffer was current. Now the handler captures `ownBuffer` once and threads it through both `appendEvent(chatId, chunk, ownBuffer)` and `finishStreamIfCurrent(chatId, ownBuffer)` so the identity check lives at the registry boundary. (#194)
- **`{{inputs.x}}` interpolation in agent action prompts.** Friday workspaces (which exclusively use agent actions) previously saw literal `{{inputs.description}}` in the prompt while the resolved values sat in an appended `## Input` block. Placeholders are now interpolated uniformly across both action types. (#199)
- **`cancelSession` no longer wedges when an engine ignores `AbortSignal`.** Workspace agents that make blocking calls without piping the AbortSignal through (e.g., `fetch` with no `signal:`) used to pin cascade-stream's in-flight slot forever — every subsequent trigger of the same signal was rejected as `skipped-duplicate` until daemon restart. New `awaitWithAbort` helper races `engine.signal()` against the effective AbortSignal, releasing the slot within ~1 tick on cancel; the orphaned `engine.signal` continues in the background with a tail handler that logs settlement and prevents `UnhandledPromiseRejection`. A `finalized` flag short-circuits late events so they don't land in JetStream after `session:complete`. (#202)
- **MCP cold-start probe timeout on first install.** The 5-second foreground probe was too tight for `npx` / `uvx` MCP servers on first run — npm cold-downloaded the package in 5–6s, timing out the user's first click while the second succeeded against cached npm. The new prewarm + cache pipeline (see Added) eliminates the cold-click failure mode. (#195)
- **FSM-path MCP servers now receive registry `platformEnv` variables.** The code-execution path through `runtime.ts` was handing raw workspace MCP server configs to the FSM engine without applying registry-owned `platformEnv` (`MCP_ENABLE_OAUTH21`, dummy `GOOGLE_OAUTH_CLIENT_ID` / `SECRET`, `WORKSPACE_MCP_STATELESS_MODE`). Cron-triggered LLM actions spawned `workspace-mcp` without those vars, so it ran in native-OAuth mode and rejected every bearer-authed request — meanwhile chat (via `discoverMCPServers`, which does apply `platformEnv`) worked fine. Mirrors the same pattern used in `executeCodeAgent`. (#196)
- **MCP subprocess registry evicts stale children on spec drift.** `sharedMCPProcesses.acquire` was first-spawner-wins: whichever caller hit `acquire(serverId, …)` first locked their `spec.env` for the daemon's lifetime, so a cached subprocess spawned by a buggy code path served stale config until restart. Each cached entry now carries a stable hash of `{command, args, env}`; on mismatch the registry SIGTERMs the child, waits up to 2s, SIGKILLs as fallback, and awaits full exit before respawn (mandatory because `workspace-mcp` binds fixed ports — a racing respawn through `TIME_WAIT` is the exact failure this registry exists to avoid). (#196)
- **MCP registry search tolerates unknown `registryType`.** `/api/mcp-registry/search` was returning 502 for any query matching a server with a `registryType` outside the hardcoded Zod enum (e.g., `nuget`); upstream defines the field as an open string, so a single unrecognised value tanked the entire response. Each entry is now parsed independently — malformed ones are dropped with logging — and the response shape gains a `dropped` count for telemetry. (#191)
- **Bundled-agent lookup now uses `Set.has` instead of the `in` operator.** `in` walks the prototype chain, so `agent: "toString"` / `"constructor"` / `"hasOwnProperty"` silently passed validation — the same shape of silent-failure bug the four-layer phantom-ref defence (above) prevents. Switched to a `Set` built from `Object.keys()` for safe own-property lookup. (#201)
- **`.env` quote handling for API keys and secrets.** The Settings UI's `PUT /api/config/env` used `@std/dotenv`'s `stringify`, which wrapped any value containing non-word chars (e.g., `-` in `sk-ant-foo`) in single quotes. The launcher read those values raw and passed them to spawned services, so agents inherited `ANTHROPIC_API_KEY='sk-ant-foo'` with literal quotes, breaking authentication. Values are now serialised unquoted unless truly necessary (whitespace, `#`, `$`, leading quote, newline, backslash); the launcher strips one layer of surrounding matching quotes on read so existing hand-quoted or bug-quoted files continue working. - **`.env` parsing now trims trailing `\r` on CRLF files.** Both Rust (`parse_env_lines`) and the Go launcher (`loadDotEnv`) strip trailing `\r` so Windows-edited `.env` files parse correctly; internal CR is preserved. (#214)
- **Platform env vars refresh on every install / update.** Previous `write_env_file` only ran when the API Keys step rendered, so `FRIDAY_JETSTREAM_STORE_DIR` and other platform defaults wouldn't propagate on quiet updates. New `ensure_platform_env_vars` Tauri command runs unconditionally in `Extract.svelte` with add-if-missing semantics — never overwrites, only appends new keys. (#214)
- **`atlas migrate` reads `.env` for `FRIDAY_PORT_FRIDAY` daemon-probe.** Was hardcoded to 8080, broken in installer mode where the daemon binds to a configurable port. Now reads `.env` via `@std/dotenv`. (#214)
- **`studio-release` shell parser absorbed trailing ellipsis.** Under some locales bash treats UTF-8 continuation bytes as identifier characters, so `$publish_wf…` was parsed as including the ellipsis bytes, tripped `set -u`, and failed the publish workflow. Bracing as `${publish_wf}…` stops the parser at the `}`. (#190)

## [0.1.1] - 2026-05-01

### Added

- Workspace-scoped skill assignment tools (`assign_workspace_skill` / `unassign_workspace_skill`) for the workspace-chat agent. Calls `SkillStorage` directly, validates `@namespace/name` refs via `SkillRefSchema`, and is idempotent. Assignments take effect on the next chat turn because `resolveVisibleSkills` reads the assignment table every turn. System prompt and `workspace-api` skill reference updated to distinguish ephemeral `load_skill` from persistent workspace assignment. (#114)

### Fixed

- Hung MCP servers no longer block workspace chats. `Promise.allSettled` replaces the serial connect loop so one stuck server cannot stall the others, and hard timeouts are enforced at every phase: 4s on the HTTP reachability probe, 20s on `createMCPClient` + `listTools` handshake, and a 15-minute ceiling on `callTool` invocation. Timed-out servers are silently dropped; structured `operation: "mcp_timeout"` telemetry (with `serverId`, `phase`, `durationMs`) is logged for future circuit-breaker decisions. The previous retry on hang was removed — one shot, one timeout, move on. (#116)
- Four macOS launcher / tray bugs (#113):
  - Autostart plist now points at the bundle ID the installer actually writes (`ai.hellofriday.studio`), not the never-installed `-launcher` suffix. LaunchServices no longer fails autostart and the tray "Restart" action with `LSCopyApplicationURLsForBundleIdentifier`. Existing broken installs self-heal on next launcher start.
  - Tray "Restart all" no longer paints the menubar `Error` for the few seconds children take to come back up. A new restart-grace window (active during the in-flight restart and for `bucketFailGraceWindow` after) suppresses the transient `AnyFailed` bucket.
  - Tray "Restart all" no longer terminates the supervisor watchdog. `RestartAll` now calls `processRunner.RestartProcess` per service in `startOrder`, which keeps `pendingRestarts > 0` and prevents process-compose's `runner.Run` loop from exiting "Project completed" between the stop and start passes. As a side benefit, four of five services stay running at any instant during a user-initiated restart.
  - The user's "Start at login" disabled choice now survives launcher restarts. `isAutostartStale()` on darwin matches the Windows contract: stale iff the registered value is both non-empty and mismatched. An absent plist means user-disabled and is left alone.
  - The cold-start grace and post-restart grace are unified behind a single `bucketFailGraceWindow` constant (90s) that covers the readiness-probe budget (`InitialDelay=2s + FailureThreshold=30 × PeriodSeconds=2s = 62s`), so legitimately slow first-pass readiness (the friday daemon's ~24s workspace scan + skill bundle hashing + cron registration on cold start) no longer flips the bucket red mid-startup.
