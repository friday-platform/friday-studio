# 2026-04-24 — MCP Custom Server Addition (Team Lead Session)

## CLAUDE.md Hard Rules — Recurring Violations

- **Two teammates independently used `as Record<string, unknown>`** after `typeof x === "object" && x !== null && !Array.isArray(x)` checks. The correct pattern is a small `isPlainObject(v: unknown): v is Record<string, unknown>` type guard. It satisfies both TypeScript and the style guide without duplication.
- **Non-null assertion `names[0]!`** was used after an explicit `names.length === 0` early-return. TypeScript control-flow does not narrow `array[0]` after length checks in Deno's strict mode. Destructuring `const [first] = names` plus `if (!first) return` is the clean fix.

## Codebase Quirks That Confused Teammates

- **Deno check pulls transitive deps.** Running `deno check` on a single file can surface 15+ pre-existing errors in unrelated packages. Always filter by file path before declaring a task has type errors.
- **`tools/agent-playground` queries directory is excluded from root `deno check`.** Effective type-check gate for playground code is `cd tools/agent-playground && npx tsc --noEmit` (or `svelte-check` for `.svelte` files).
- **Svelte 5 interprets curly braces inside attribute strings as expression tags.** Placeholder text with literal `{}` (e.g., JSON examples in placeholders) must be avoided or escaped.

## Patterns Worth Documenting

- **Mocking `@atlas/mcp` for integration tests.** A `vi.hoisted` mock that returns a synthesized tool per server ID is a clean pattern for playground route tests that need to assert the 200 success path without spawning real MCP servers.
- **Extracting shared helpers from route handlers.** Jinju extracted `createLinkProvider` from the install route so both `/install` and `/custom` could reuse the Link POST logic. This keeps route handlers focused on request/response orchestration.
