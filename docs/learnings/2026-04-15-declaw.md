## Observations worth remembering

- **Untracked trees break `git mv`.** The whole `packages/bundled-agents/src/web/`
  subtree was untracked on declaw. Storm's D1 rename discovered that `git mv
  steel-utils.ts agent-browser-utils.ts` fails because the source is not in the
  index. Plain `mv` + `git add` is the path. Worth flagging in CLAUDE.md or
  the committing skill if this recurs.
- **Deletion with no diff is valid.** Po's D3 deleted `credential.ts` but the
  file was untracked, so there was literally no commit to make. The correct
  answer was "no commit needed, delete is on disk." A teammate newer to the
  flow could easily get stuck trying to force a commit here.
- **Module-level preflight doesn't run during `evals list`.** Luka noticed that
  `deno task evals list` is static discovery — the top-of-file
  `execFileAsync("agent-browser", ["--version"])` call never fires during
  listing. Only at actual run time. Useful for designing future preflight gates.
- **`deno task typecheck` includes transitive workspace files.** Every
  teammate in this session reported "my files are clean but the workspace has
  unrelated errors." The review flow accommodated this by checking the diff
  against changed files only, then escalating the workspace-level gate as a
  separate concern. Possible future skill gap: a "typecheck-my-diff" helper
  that scopes to changed files.
- **Snapshot test failure messages should include the actual hash.** Ferox's
  D5 snapshot test emits the computed hash in the `expect().toBe(EXPECTED)`
  failure so the developer can copy-paste the new constant straight from
  output. Worth stealing for other prompt snapshot tests.
- **Heavy-cap rule holds up.** Ferox (D5) and Jinju (D2) each did one heavy
  task and shut down clean. Po doing light-then-medium (D3+D4) also worked —
  mixed-weight rotation is fine as long as the cumulative count respects the
  tighter cap.
