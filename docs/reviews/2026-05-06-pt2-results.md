## Architectural decisions baked in

1. **External = `generateObject` to a judge agent (not delegate proper).** B7 took the simpler runtime path. Tradeoff: no auto Phase 8 budgets / Phase 11 provenance / on-demand `artifacts_get`. Tracked as a follow-up — revisit if a workload pays meaningful judge-input-token cost.
2. **Refs-not-bytes handoff to the judge.** A2's text-lift means large tool results are already artifacts when the action emits. Judge handoff carries `resultArtifactId` + summary for lifted results; small results stay inline. Cost scales with what the judge inspects.
3. **Auto-detect never returns `external`.** Only `skip` or `self`. External fires only on explicit `validate: external` or `validation.default: external`. Most workloads pay zero validation cost.
4. **case "agent" → type: llm gets validation via the orchestrator** (B4). Pre-fix: only case "llm" inline got validated. Post-fix: identical behavior regardless of FSM action shape. Closes the asymmetry exposed by chat_HO8akAMxe4's auto-triage refactor.
5. **`record_validation` is auto-injected only when decision === self AND no outputType.** E1's fix preserves the structured-output guarantee. Authors who want self-verdict on structured actions split into two FSM steps.
6. **`validate:` strategies and per-strategy verdict mechanics:**
   - `skip` → runtime synthesizes `{ strategy: "skip", skipReason }`. No LLM.
   - `self` → skill loaded into action's prompt; LLM calls `record_validation` (free-form actions only). Verdict from tool args.
   - `external` → judge spawn. Verdict from `generateObject` output structured per `validation-verdict`.

## Learnings worth carrying forward

- **Concurrent sub-agents in the same worktree race on shared files.** Disjoint targets (A2 + B1 + C1) work; shared targets (B2 + C1, B3 + C2) caused test runs to see each other's WIP and produce misleading reports. Mitigation: serialize sub-agents whose target files overlap.
- **Live-test exposed a class of bugs no unit test caught.** B6's `toolChoice: "auto"` change passed all unit tests but broke a real production action. Lesson: when changing tool-injection mechanics for structured-output actions, run a real workspace.yml job before commit. Mocked LLMs can't catch a model "deciding to emit prose instead of calling complete."
- **NATS data inspection without a daemon: copy + standalone nats-server.** `~/.atlas/jetstream/jetstream/$G/streams/<STREAM>` is the on-disk data. Copy to safe dir, run `nats-server` against `store_dir: /tmp/copy/jetstream`, then `nats stream get` etc. Critical: do NOT point `nats-server` at the live data dir — it relocates the layout.
- **Misleading flag semantics surface when filters tighten.** `user-invocable: false` had been on 5 authoring skills since initial commit, with the runtime blind to it. Pt2 introduced the filter; the filter then exposed the misconfiguration. Pattern: when adding a runtime check on a frontmatter / config flag, audit existing call sites before landing.
- **Sub-agent reports are sometimes wrong about cause.** C1 reported a "pre-existing" test failure that was actually B2's WIP visible mid-flight. E1 imported a non-existent symbol assuming its companion would create it (E2 did, but the timing was tight). Pattern: trust the change set + reproduce locally before pinning blame.
